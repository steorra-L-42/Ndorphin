3. Docker 컨테이너 실행

Shell
복사
# 기본 커맨드 # -d: detach mode (백그라운드로 실행됨) # -p: 호스트에 연결된 컨테이너의 특정 포트를 외부에 노출하기 위해 사용 # docker run -d -p [호스트 port]:[컨테이너 port] [DockerHub ID]/[이미지명] # : Docker Image를 실행한 형태인 컨테이너로 만드는 명령어 # --name : 컨테이너 이름 설정 docker run -d -p 8080:8080 {도커허브 ID}/{도커허브 레포} ex) docker run -d --name spring_container -p 8080:8080 qoridhc/ndolphin
​
이미 한 번 이미지를 컨테이너화 했을 때, docker run 명령어를 다시 사용하면 안 됨
docker run의 경우 Docker Hub에서 해당 이미지를 pull & 컨테이너화하는 작업이기 때문에, docker ps -a를 사용해보면 같은 이미지가 여러 개 생성되어 있는것을 확인 할 수 있음
> 따라서 이미 컨테이너가 존재한다면 docker start [컨테이너명 or ID] 로 실행

1. MySql 컨테이너 실행

JavaScript
복사
docker run --name mysql-container -e MYSQL_ROOT_PASSWORD=my-secret-pw -d -p 3306:3306 mysql:latest
​
-name mysql-container: 컨테이너의 이름을 mysql-container로 지정
e MYSQL_ROOT_PASSWORD=my-secret-pw: MySQL 루트 비밀번호를 my-secret-pw로 설정
d: 컨테이너를 백그라운드에서 실행
p 3306:3306: 호스트의 3306 포트를 컨테이너의 3306 포트에 매핑

-name mysql-ndolphin: 컨테이너 이름을 mysql-ndolphin으로 지정합니다.

e MYSQL_USER=ssafy: 사용자 이름을 ssafy로 설정합니다.

p 3306:3306: 호스트의 3306 포트를 컨테이너의 3306 포트에 매핑합니다.
d: 백그라운드에서 실행합니다.

2. nginx 설정

2.1 sites-avaliable & sites-enabled

한 웹서버에서 여려가지 웹 서비스를 다른 도메인으로 운영하기 위한 폴더
sites-available에 각 도메인 별 설정을 저장해두고 sites-enabled에 심볼릭 링크를 설정하거나 해제하는 식으로 손쉽게 웹서비스를 실행, 중단 시킬 수 있음
sites-available
Nginx 서버에 구성된 모든 사이트의 설정 파일을 저장하는곳
단지 서버에 필요한 설정을 모아둔 폴더일 뿐 실제 활성화 된건 아님
sites-enabled
Nginx에서 활성화 하여 실제 사용중인 설정파일
sites-available에서 심볼릭 링크를 통해 연결되어 사용됨

2.2 nginx.conf

복사

젠킨스 컨테이너 띄우기

JavaScript
복사
sudo docker run -d -p 9090:8080 -v /jenkins:/var/jenkins_home --name jenkins -u root jenkins/jenkins:lts
​
d : 컨테이너를 백그라운드에서 실행
p 9090:8080 : 컨테이너 외부와 내부 포트를 포워딩, 좌측이 호스트 포트, 우측이 컨테이너 포트
v /jenkins:/var/jenkins_home : 도커 컨테이너의 데이터는 기본적으로 컨테이너가 종료되면 휘발됨. 따라서 도커 컨테이너의 데이터를 보존하기 위한 여러 방법이 존재하는데, 그 중 한 방법이 볼륨 마운트. 이 옵션을 사용하여 젠킨스 컨테이너의 /var/jenkins_home 이라는 디렉토리를 호스트의 /jenkins 와 마운트하고 데이터를 보존할 수 있다.
-name jenkins : 도커 컨테이너의 이름을 설정
u root : 컨테이너가 실행될 리눅스의 사용자 계정을 root 로 명시

Jenkins Pipeline

JavaScript
복사
pipeline { agent any environment { DOCKER_HUB_CREDENTIALS = credentials('DockerHub Id') // Jenkins에 저장된 Docker Hub 자격 증명 ID EC2_HOST = "ubuntu@i11d207.p.ssafy.io" SSH_KEY = credentials('ec2-ssh-key') // Jenkins에 저장된 SSH 키 ID DOCKER_IMAGE_NAME = "qoridhc/ndolphin" } stages { stage('Clone') { steps { echo 'Cloning repository...' git branch: 'realese', credentialsId: 'gitlab-personal-api', url: 'https://lab.ssafy.com/s11-webmobile2-sub2/S11P12D207.git' echo 'Clone finished!' } } stage('Docker Build & Push') { steps { script { echo 'Building Docker image...' dir('backend/ndolphin') { // gradlew 파일에 실행 권한 추가 sh 'chmod +x gradlew' // 실행 권한 추가 // Gradle 빌드 실행 sh './gradlew clean build --warning-mode all' } echo 'Logging into Docker Hub...' sh "echo ${DOCKER_HUB_CREDENTIALS_PSW} | docker login -u ${DOCKER_HUB_CREDENTIALS_USR} --password-stdin" echo 'Pushing Docker image to Docker Hub...' sh "docker push ${DOCKER_IMAGE_NAME}:latest" } } } stage('Deploy to EC2') { steps { script { echo 'Deploying to EC2...' sshagent(['ec2-ssh-key']) { sh """ ssh -o StrictHostKeyChecking=no ${EC2_HOST} << EOF docker login -u ${DOCKER_HUB_CREDENTIALS_USR} -p ${DOCKER_HUB_CREDENTIALS_PSW} docker pull ${DOCKER_IMAGE_NAME}:latest docker-compose stop # 기존 컨테이너 종료 docker-compose up -d # 백그라운드에서 컨테이너 실행 EOF """ } echo 'Deployment finished!' } } } } }
